# Fixture Files System Proposal

## Overview

This proposal outlines an elegant solution for making fixture files available in both Node.js and browser test environments, following the same pattern used for migration SQL files.

## Design Pattern

The solution follows the existing pattern used for Drizzle migration files:
- **Browser**: Files are stored as string variables in a TypeScript module (like `drizzleFiles.ts`) that gets bundled into JavaScript
- **Node.js**: Files are copied from the fixtures directory to the project directory using `fs`

## Browser Flow Explained

### Build Time (Bundle Creation)
1. **Fixture files are embedded as string variables** in `fixtureFiles.ts`:
   ```typescript
   export const fixture_schema_with_ids_json = `{ "name": "Test Schema", ... }`
   ```
2. **This TypeScript file gets bundled** into the JavaScript bundle by Vite/webpack
3. **The string variables are now part of the JavaScript bundle** - they're just regular JavaScript string constants

### Runtime (Test Execution)
1. **When `setupFixtureFiles()` is called**, it imports the bundled `fixtureFiles` module:
   ```typescript
   import { fixtureFiles } from './fixtureFiles'
   // fixtureFiles is now an object with string values, already in memory
   ```
2. **The function reads the string content** from the bundled variables (already in memory, no file I/O needed)
3. **Writes the strings to OPFS** using `BaseFileManager.saveFile()`:
   ```typescript
   await BaseFileManager.saveFile('/app-files/schema-with-ids.json', fixtureFiles['schema-with-ids.json'])
   ```
4. **Files are now available in OPFS** and can be read by tests using normal file paths

### Key Points
- **No file I/O at build time** - files are just string constants in the bundle
- **Files are written to OPFS at runtime** when `setupFixtureFiles()` is called
- **This matches the migration file pattern** - see `src/browser/db/Db.ts:copyDrizzleFiles()` which does the same thing

### Example Flow

**1. Build Time:**
```typescript
// fixtureFiles.ts (generated by build script)
export const fixture_schema_with_ids_json = `{"name": "Test Schema", ...}`

// This gets bundled into JavaScript:
// bundle.js contains: const fixture_schema_with_ids_json = "{\"name\": \"Test Schema\", ...}"
```

**2. Runtime - Test Setup:**
```typescript
// In test file
beforeAll(async () => {
  // This imports the bundled module (strings are already in memory)
  await setupFixtureFiles(['schema-with-ids.json'])
})
```

**3. Runtime - setupFixtureFiles() executes:**
```typescript
// setupFixtureFiles.ts
import { fixtureFiles } from './fixtureFiles' // Already bundled, just imports object

async function setupFixtureFilesBrowser(fixtureNames: string[]) {
  // fixtureFiles is an object: { 'schema-with-ids.json': '{"name": "Test Schema", ...}' }
  const content = fixtureFiles['schema-with-ids.json'] // Gets string from memory
  
  // Write string to OPFS (this is the only file I/O)
  await BaseFileManager.saveFile('/app-files/schema-with-ids.json', content)
}
```

**4. Runtime - Test uses file:**
```typescript
// Test can now read from OPFS
const file = await BaseFileManager.readFileAsString('/app-files/schema-with-ids.json')
```

### Comparison with Migration Files

This is **exactly** how migration files work:

```typescript
// src/browser/db/drizzleFiles.ts (bundled)
export const migrationSql_0000_married_malice = `CREATE TABLE ...`

// src/browser/db/Db.ts:copyDrizzleFiles() (runtime)
import * as drizzleFiles from './drizzleFiles' // Bundled module
const migrationContent = drizzleFiles.migrationSql_0000_married_malice // String from bundle
await BaseFileManager.saveFile(`${filesDir}/db/0000_married_malice.sql`, migrationContent)
```

### 2. Unified Setup Function

Create `__tests__/test-utils/setupFixtureFiles.ts` with a single function that works in both environments:

```typescript
/**
 * Makes fixture files available in the test environment
 * 
 * Browser: Writes files to OPFS using BaseFileManager
 * Node.js: Copies files from __tests__/__fixtures__ to project directory
 * 
 * @param fixtureNames - Array of fixture file names (e.g., ['schema-with-ids.json'])
 * @param targetDir - Optional target directory (defaults to project root or /app-files in browser)
 * @returns Promise that resolves when all files are available
 * 
 * @example
 * ```typescript
 * beforeAll(async () => {
 *   await setupFixtureFiles(['schema-with-ids.json', 'schema-without-ids.json'])
 * })
 * ```
 */
export async function setupFixtureFiles(
  fixtureNames: string[],
  targetDir?: string
): Promise<void> {
  const isNodeEnv = typeof window === 'undefined'
  
  if (isNodeEnv) {
    // Node.js: Copy files from fixtures directory
    await setupFixtureFilesNode(fixtureNames, targetDir)
  } else {
    // Browser: Write files to OPFS
    await setupFixtureFilesBrowser(fixtureNames, targetDir)
  }
}
```

### 3. Node.js Implementation

```typescript
async function setupFixtureFilesNode(
  fixtureNames: string[],
  targetDir?: string
): Promise<void> {
  const path = await import('path')
  const fs = await import('fs')
  const { fileURLToPath } = await import('url')
  
  // Get fixtures directory
  const testUtilsDir = path.dirname(fileURLToPath(import.meta.url))
  const fixturesDir = path.join(testUtilsDir, '..', '__fixtures__')
  
  // Determine target directory
  // If targetDir is provided, use it
  // Otherwise, use the project directory from config
  const { PathResolver } = await import('@/node/PathResolver')
  const pathResolver = PathResolver.getInstance()
  const projectRoot = process.cwd()
  
  const finalTargetDir = targetDir 
    ? path.resolve(targetDir)
    : projectRoot
  
  // Ensure target directory exists
  if (!fs.existsSync(finalTargetDir)) {
    fs.mkdirSync(finalTargetDir, { recursive: true })
  }
  
  // Copy each fixture file
  for (const fixtureName of fixtureNames) {
    const sourcePath = path.join(fixturesDir, fixtureName)
    const targetPath = path.join(finalTargetDir, fixtureName)
    
    if (!fs.existsSync(sourcePath)) {
      throw new Error(`Fixture file not found: ${fixtureName} at ${sourcePath}`)
    }
    
    // Copy file (overwrite if exists)
    fs.copyFileSync(sourcePath, targetPath)
    console.log(`[setupFixtureFiles] Copied ${fixtureName} to ${targetPath}`)
  }
}
```

### 4. Browser Implementation

```typescript
async function setupFixtureFilesBrowser(
  fixtureNames: string[],
  targetDir?: string
): Promise<void> {
  const { BaseFileManager } = await import('@/helpers/FileManager/BaseFileManager')
  const { fixtureFiles } = await import('./fixtureFiles')
  
  // Determine target directory
  // Default to /app-files (from config) or provided targetDir
  const finalTargetDir = targetDir || '/app-files'
  
  // Ensure target directory exists
  await BaseFileManager.createDirIfNotExists(finalTargetDir)
  
  // Write each fixture file to OPFS
  for (const fixtureName of fixtureNames) {
    const content = fixtureFiles[fixtureName]
    
    if (!content) {
      throw new Error(`Fixture file not found: ${fixtureName}. Available fixtures: ${Object.keys(fixtureFiles).join(', ')}`)
    }
    
    const targetPath = `${finalTargetDir}/${fixtureName}`
    await BaseFileManager.saveFile(targetPath, content)
    console.log(`[setupFixtureFiles] Wrote ${fixtureName} to ${targetPath}`)
  }
}
```

## Alternative: Relative Path Helper

To simplify path resolution, add a helper:

```typescript
/**
 * Gets the path to a fixture file in the current environment
 * @param fixtureName - Name of the fixture file
 * @param baseDir - Optional base directory (defaults to project root or /app-files)
 * @returns Path to the fixture file
 */
export function getFixturePath(fixtureName: string, baseDir?: string): string {
  const isNodeEnv = typeof window === 'undefined'
  
  if (isNodeEnv) {
    const path = require('path')
    return path.join(baseDir || process.cwd(), fixtureName)
  } else {
    return `${baseDir || '/app-files'}/${fixtureName}`
  }
}
```

## Benefits

1. **Single API**: One function works in both environments
2. **Consistent Pattern**: Follows existing migration file pattern
3. **Type Safety**: Fixture names can be typed/validated
4. **Build-Time**: Fixtures are embedded at build time for browser
5. **Flexible**: Supports custom target directories
6. **Simple Usage**: Easy to call from test setup

## Implementation Steps

1. Create `__tests__/test-utils/fixtureFiles.ts` (manually or via script)
2. Create `__tests__/test-utils/setupFixtureFiles.ts` with unified function
3. Create build script or Vite plugin to generate `fixtureFiles.ts`
4. Update test files to use `setupFixtureFiles()`
5. Add helper function `getFixturePath()` for convenience

## Considerations

- **File Size**: Large fixture files might bloat the browser bundle
- **Sync**: Need to ensure `fixtureFiles.ts` stays in sync with `__fixtures__` directory
- **Path Resolution**: May need to handle different project structures
- **Cleanup**: Consider cleanup after tests (optional)
